# Article Digest

把文章整理成适合日报/晨报/简报分发的中文内容。

## 适用场景

- 单篇文章整理成“日报”格式
- 多篇 RSS / 博客 / 新闻聚合成一份简报
- 把长文压缩成适合 Discord、Telegram、笔记软件转发的内容
- 把技术文章整理成更像内部晨报或行业观察的格式

## 输入要求

尽量收集这些信息：

- 标题
- 日期
- 来源
- 原始链接
- 正文或可抓取正文
- 可选：面向对象（技术/投资/内部同步/泛读者）

如果正文抓取不完整，也可以输出，但要明确标注：

> 以下整理基于可抓取正文片段。

## 默认输出目标

输出应满足：

- 中文
- Markdown 分级标题清晰
- 核心结论先行
- 保留原始链接
- 不堆砌原文，不逐段翻译
- 不编造原文没有的信息
- 保留关键数字、公司名、模型名、版本号、时间点
- 避免空话，比如“值得关注”“引发热议”这类无信息量描述

## 单篇文章：默认日报格式

使用下面结构。

```markdown
# {{日报标题}}

## 日期
{{发布日期}}

## 来源
{{来源名称}}

## 原始链接
<{{原始链接}}>

## 今日核心
用 1-3 句话写出最重要的结论：发生了什么、为什么重要、影响什么。

## 一、发生了什么
- 关键事实 1
- 关键事实 2
- 关键事实 3

## 二、核心信息提炼
### 1. {{子主题}}
简洁说明

### 2. {{子主题}}
简洁说明

### 3. {{子主题}}
简洁说明

## 三、为什么值得关注
- 对行业/产品/工程/商业的直接影响
- 如果只是观点文，要说明这是作者判断，不是确定事实

## 一句话点评
用一句话收束，像日报末尾的结论句。
```

## 多篇合辑：日报/晨报格式

当用户要“做成日报”且涉及多篇文章时，使用下面结构。

```markdown
# AI / 科技日报｜{{主题}}

## 今日要点
- 要点 1
- 要点 2
- 要点 3

## 1. {{文章标题 A}}
### 核心
1-2 句话总结

### 关键信息
- 点 1
- 点 2
- 点 3

### 原始链接
<{{链接}}>

## 2. {{文章标题 B}}
...

## 今日观察
给出横向判断：这些新闻共同说明了什么。
```

## 写作规则

### 1) 先结论，后展开
不要先铺背景。第一屏就要让人知道这篇东西讲什么。

### 2) 事实与判断分开
- **事实**：文章明确写了什么
- **判断**：作者观点或你的提炼

如果原文本身就是评论/周报，也要说明“这是评论型内容”。

### 3) 尽量压缩成“信息块”
优先整理成：

- 新产品 / 新模型 / 新发布
- 关键参数 / 数据 / 能力
- 影响范围
- 风险 / 限制 / 未确认点

### 4) 不逐段改写原文
日报不是翻译稿。要去冗余、做抽象、保留最有用的信息。

### 5) 技术文要保留硬信息
尤其保留：

- 参数规模
- 速度/吞吐/延迟
- 接口/协议
- 芯片/模型/框架名称
- 时间和版本

### 6) 链接必须保留
原始链接单独一节，格式优先用：

```markdown
<https://example.com>
```

### 7) Discord / WhatsApp 不用表格
统一用标题 + 列表。

## 风格档位

### A. 标准日报版（默认）
- 简洁
- 信息密度高
- 适合日常分发

### B. 内部晨报版
额外补一节：

```markdown
## 对我们意味着什么
- 机会
- 风险
- 可跟进事项
```

### C. 技术跟踪版
额外补一节：

```markdown
## 技术关注点
- 架构 / 协议 / 模型能力
- 工程实现难点
- 与现有方案相比的差异
```

## 快速判定模板

### 如果是“产品发布文”
重点写：
- 发布了什么
- 和上一代比强在哪
- 关键参数
- 潜在应用

### 如果是“行业评论文”
重点写：
- 作者主判断
- 支撑案例
- 背后趋势
- 你是否认同（可选）

### 如果是“技术教程/架构文”
重点写：
- 解决什么问题
- 方法核心是什么
- 适用场景
- 限制和代价

## 交付标准

一篇合格的日报整理，应满足：

- 标题一眼看懂主题
- 前 3 行能读懂结论
- 数字、名字、链接齐全
- 没有明显废话
- 不需要读原文也能理解七八成
- 读完后知道要不要点原文

## 推荐默认行为

当用户说：

- “整理成日报”
- “写成晨报格式”
- “给我一个简报版”
- “按分级标题整理一下”

默认按 **单篇文章：默认日报格式** 输出，除非用户明确要求别的样式。
